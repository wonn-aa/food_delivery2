# Лабораторная работа №2: Обработка потоковых данных с Apache Kafka и Airflow

## Описание проекта
Данный проект реализует систему потоковой обработки данных для условной **Службы доставки еды**. Система включает в себя генератор событий, брокер сообщений Kafka для их транспортировки, скрипт-потребитель для валидации данных и оркестратор Airflow для управления регламентными задачами.

## Задачи работы
- Развернуть Apache Kafka в режиме **KRaft** (без Zookeeper).
- Разработать масштабируемый скрипт поставщика данных (**Kafka Producer**) с соблюдением принципов **SOLID**.
- Разработать скрипт потребителя (**Kafka Consumer**) с механизмом валидации входящих JSON-сообщений.
- Интегрировать процесс в **Apache Airflow** для автоматизации отчетности.

---

## Архитектура решения

### 1. Генерация данных (`generator.py`)
Согласно принципу единственной ответственности (**Single Responsibility**), логика формирования данных вынесена в отдельный класс `FoodDeliveryGenerator`. 
*   **Отражаемые сущности:** Рестораны, меню, клиенты, курьеры, заказы, отзывы.
*   **Формат:** JSON.

### 2. Поставщик данных (`producer.py`)
Скрипт имитирует работу мобильного приложения или сайта. Он генерирует случайные заказы, выводит их в консоль и отправляет в топик `food_delivery_topic`.

### 3. Потребитель данных (`consumer.py`)
Скрипт в реальном времени считывает сообщения из Kafka. 
*   **Валидация:** Проверяет корректность параметров (например, `total_price > 0`).
*   **Обработка ошибок:** Если сообщение не проходит валидацию, в консоль выводится текст `NOT VALID` и содержимое объекта.

### 4. Оркестрация (`dags/food_delivery_dag.py`)
Реализован DAG в **Apache Airflow**, который отвечает за ежедневный запуск аналитических задач на основе данных, накопленных в Kafka.

## Стек технологий
*   **Инфраструктура:** WSL2 (Ubuntu), Apache Kafka 4.2.0 (KRaft mode).
*   **Язык разработки:** Python 3.11+.
*   **Библиотеки:** `kafka-python-ng`, `json`, `random`.
*   **Оркестрация:** Apache Airflow 2.10.4.

## Инструкция по запуску

### 1. Запуск Kafka (KRaft)
# Форматирование хранилища (выполняется один раз)
bin/kafka-storage.sh format --standalone -t $UUID -c config/kraft/server.properties

# Запуск сервера
bin/kafka-server-start.sh config/kraft/server.properties

### 2. Запуск скриптов обработки
В разных терминалах выполните:

# Запуск потребителя
python3 consumer.py

# Запуск поставщика
python3 producer.py

### 3. Запуск Airflow

airflow webserver --port 8080
airflow scheduler

### Результат работы
Система успешно демонстрирует полный цикл работы с данными:
Генерация: Создание JSON-сообщения.
Передача: Доставка через брокер Kafka с минимальной задержкой.
Обработка: Валидация данных "на лету" на стороне Consumer.
Мониторинг: Визуализация выполнения задач в интерфейсе Airflow.
